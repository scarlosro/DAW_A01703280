Preguntas



  • ¿Por qué es importante hacer un session_unset() y luego un session_destroy()?
    Por que destruye toda la información guardada y libera todas las variables de la sesión[1]


  • ¿Cuál es la diferencia entre una variable de sesión y una cookie?
    Una cookie, los datos asociados a una sesión se almacenan en el servidor y nunca en el cliente. [2]


  • ¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?
    Se verifica si ya existe la foto, y en dado caso que ya puede generar una tecnica para subir la foto como un incrementador


  • ¿Qué es CSRF y cómo puede prevenirse?
    Para que se pueda llevar a cabo un ataque mediante CSRF, en primera instancia, el atacante debe conseguir vulnerar e ingresar código HTML en un determinado servidor que podemos llamar “A”.Por otro lado, la víctima establece una conexión legÍtima con una aplicación web en otro servidor, que llamaremos “B”.La víctima accede a la aplicación web donde se encuentra el código introducido por el atacante.El navegador de la víctima realiza una petición contra la aplicación del servidor web “B” sin que el usuario se entere
    Se puede prevenir con: Modo incognito, cerrar sesión, no guardar formularios.[3]
